IT Project Team56 OOSD-Questify : Coding & Review Workflow

Created by Chang Chang on Sep 21, 2025

Task Workflow

  1. Tasks created & refined in Jira

  2. Assigned to a team member, linked to a branch

  3. Status flow: To Do → In Progress → In Review → Done

  4. Every Jira ticket must link to a PR in GitHub

Code Review Process

  • All changes go through Pull Requests (PRs)

  • Reviewer assignment:

    • Backend → reviewed by another backend dev or CI/CD lead

    • Frontend → reviewed by another frontend dev

Review Checklist

  • Code compiles & tests pass locally

  • Meets coding standards & style

  • Functions/methods small & clearly named

  • Security/auth checks included

  • UI covers loading/error/empty states

  • Documentation updated

  • No secrets or sensitive data

Merge Policy

  • Squash & merge after:

    1. CI pipeline passes

    2. At least one approval from another team member

Testing & CI/CD

  • Unit Tests: backend APIs, frontend components

  • Integration Tests: REST endpoints + DB consistency

  • GitHub Actions: runs on every PR (Lint → Test → Build)
