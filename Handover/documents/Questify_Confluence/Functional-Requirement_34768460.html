IT Project Team56 OOSD-Questify : Functional Requirement

Created by Chang Chang, last modified on Oct 24, 2025

FR-001: User Authentication

  • FR-001.1 The system shall allow account creation for current students using university email domain only.

  • FR-001.2 The system shall allow users to log in and log out.

  • FR-001.3 The system shall allow users to reset their password.

FR-002: Roles & Permissions

  • FR-002.1 The system shall support roles: Student, Teaching team, Administrator.

  • FR-002.2 The system shall restrict actions by role (e.g., only Admin/Teaching team can verify/reject questions).

FR-003: User Profile

  • FR-003.1 The system shall allow a student to set a profile name.

  • FR-003.2 The system shall allow a student to set a profile image.

  • FR-003.3 The system shall display a student’s attempt history on their profile.

  • FR-003.4 The system shall display a student’s post question history on their profile.

  • FR-003.5 The system shall display a student’s saved questions on their profile.

FR-004: Question Authoring (Posting)

  • FR-004.1 The system shall allow a student to write a question and answer, selecting its week and topic.

  • FR-004.2 The system shall generate an AI answer to short answer questions.

  • FR-004.3 The system shall award posting rewards when applicable.

FR-005: Question Solving

  • FR-005.1 The system shall allow a student to attempt questions.

  • FR-005.2 The system shall allow a student to save questions to a personal question list (bookmarks).

  • FR-005.3 The system shall record attempts to support progress tracking and mistake review.

FR-006: Question List & Discovery

  • FR-006.1 The system shall display a list of all questions.

  • FR-006.2 The system shall allow sorting/grouping of the list.

  • FR-006.3 The system shall allow searching for specific questions.

FR-007: Rating, Comments & Discussion

  • FR-007.1 The system shall allow students to comment on questions.

  • FR-007.2 The system shall allow students to check, reply to, and like others’ comments.

  • FR-007.1 The system shall allow students to rate questions.

FR-008: Student Engagement & Rewards

  • FR-008.1 The system shall provide a view of student activity score and ranking.

  • FR-008.2 The system shall award score for students' activity.

FR-009: Administration & Moderation

  • FR-009.1 The system shall allow Administrators/Teaching team to verify or reject submitted questions.

  • FR-009.2 The system shall provide an admin dashboard to check overall participation and identify active students.

  • FR-009.3 The system shall provide activity views across a selected period to identify when students are most engaged and which question types are most interactive.

FR-010: AI Support

  • FR-010.1 The system shall generate example answer for short answer questions.

FR-011: Content Visibility Rules

  • FR-011.1 The system shall display student made questions in public lists.

  • FR-011.2 The system shall show a pending/approved/rejected status to the author of each submitted question.

FR-012: Data Lifecycle for Accounts

  • FR-012.1 The system shall deactivate or remove accounts of past students at the end of the academic year.

  • FR-012.2 The system shall restrict new sign-ups to users with valid university emails.
