IT Project Team56 OOSD-Questify : Deployment Document

Created by Ziyi Zhang on Sep 28, 2025

1. Backend

Deployment

  • Runs on Render Web Service with PostgreSQL

  • Automated via GitHub Actions

  • Separate branches:

    • Staging → TestMainBranch

    • Production → main

Render Setup

Service (render.yaml)

services:
  - type: web
    name: questify-backend
    runtime: python3
    buildCommand: |
      cd backend && pip install -r requirements.txt
      python manage.py collectstatic --noinput
      python manage.py migrate --noinput
    startCommand: |
      cd backend && gunicorn config.wsgi:application --bind 0.0.0.0:$PORT

Database

databases:
  - name: questify-db
    databaseName: questify
    user: questify_user

Environment Variables

  • PYTHON_VERSION=3.11.0

  • SECRET_KEY (secure)

  • DEBUG=false

  • ALLOWED_HOSTS=questify-backend.onrender.com

  • DATABASE_URL (auto from Render)

Django Settings

  • Database: Uses DATABASE_URL if set, else SQLite for local dev

  • Security:

    • SECRET_KEY from env var

    • DEBUG=false in production

    • ALLOWED_HOSTS + CSRF_TRUSTED_ORIGINS → Render domain

CI/CD

  • Staging: Deploys on push to TestMainBranch

  • Production: Deploys on push to main

GitHub Secrets

  • RENDER_STAGING_DEPLOY_HOOK

  • RENDER_PRODUCTION_DEPLOY_HOOK

  • OPENAI_API_KEY (optional)

Pipeline Flow
CI tests → webhook trigger → build → migrate → Gunicorn start
